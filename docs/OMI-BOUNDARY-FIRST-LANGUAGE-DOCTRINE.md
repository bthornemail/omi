# OMI Boundary-First Language Doctrine

OMI is not symbol-first. OMI is boundary-first.

The foundational object of the system is not the glyph, token, keyword,
parser rule, or syntax tree. The foundational object is declared separation.
From separation comes segmentation, traversal, projection, geometry,
interpretation, measurement, composition, and decomposition.

## Core Statement

```text
OMI is boundary-first.
```

This is the doctrine behind the omicron name. The system begins with a
transition law between latent structure and observable projection, not with a
symbol table.

## The Space Character As Omicron

The ASCII space character (`0x20`) is not blank.

It is the first printable declaration of separation.

```text
0x00..0x1F = latent structural substrate
0x20       = projection boundary / omicron
0x21..0x7E = observable printable field
```

That makes `0x20` an omicron-like threshold operator:

- hidden -> visible
- control -> printable
- substrate -> projection
- structure -> interpretation

Space is therefore not absence. It is declared segmentation law.

## Boundary Before Symbol

Traditional systems are often organized as:

```text
symbol -> parser -> syntax tree -> semantics
```

OMI reverses that order:

```text
boundary -> segmentation -> traversal -> projection -> observable symbols
```

Meaning emerges from declared boundaries before symbols are interpreted.

That is why whitespace, indentation, line breaks, overlays, chunk boundaries,
stream scopes, and block regions all become first-class objects.

## Positional Notation And Omicron

The same principle appears in positional numbering.

```text
10 = declaration of base transition
```

The numeral `10` is not merely a sign-value symbol. It marks a transition from
one order of magnitude to the next. In that sense it is also omicron-like:
encapsulation, decomposition, and projection between regimes.

So:

```text
10   = boundary between magnitudes
0x20 = boundary between representation regimes
```

## CONS And Structural Projection

A CONS cell is not merely a pair. It is a declared segmentation boundary
between two traversable relations.

That is why OMI treats:

- car/cdr
- cdr/car
- endianness
- text direction
- traversal order
- barcode/chart
- lazy/greedy

as projection modes over the same canonical structure.

## Documents As World Models

If boundaries are primary, then documents stop being "plain text" and become
declared navigable terrains.

```text
document = declared navigable space
template  = discovery surface for a model rendition
```

A text document is therefore a 2D semantic terrain:

- line = path
- paragraph = region
- section = enclosure
- heading = coordinate anchor
- whitespace = topology
- punctuation = operator geometry
- reference = edge

## FS/GS/RS/US

The narrative plane is already structured by four resolution tiers:

| Plane | Meaning |
| --- | --- |
| FS | full world/document scope |
| GS | grouped thematic or scene region |
| RS | record / line / statement |
| US | unit / token / glyph / cell |

So Markdown, Org, AWK streams, Lisp forms, CSV tables, and even C source are
all addressable world lattices. They only look like plain text from outside.

## Parsing As Projection

In OMI, parsing is not authority. Parsing is a projection lens.

```text
canonical bytes
-> declaration scopes
-> overlay reconciliation
-> projection lenses
-> presentation surfaces
```

That separation preserves identity, replay, modality, transport, and alternate
views without collapsing them into a single parse tree.

## Narrative As World Topology

The narrative runtime is a textual world-space engine.

Reading is navigation.
Editing is world mutation.
Narrative is world traversal.
Code is executable geography.

Whitespace, indentation, and punctuation become roads, walls, portals,
coordinate transforms, and projection lenses inside the textual world model.
