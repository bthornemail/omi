# Phase 95 - OMI Transmutator Roundtrip Court

## Summary

Phase 95 proves that a small source surface can transmute into a canonical OMI
declaration, project into another notation, and return without losing declared
identity.

This is the first executable transmutator court after the observer and semantic
grounding adapter doctrines. It stays intentionally small and fixture-sized.

## Core Transmutator Law

```text
source surface
-> adapter parse / projection
-> canonical OMI declaration
-> S-P-O-M triangulation
-> target projection
-> receipt comparison
```

## Locked Phrase

> The transmutator may change notation, but it must preserve declared identity or
> explicitly emit a changed receipt.

## Supported V1 Surfaces

```text
OMI-Lisp declaration
S-P-O-M triplets
Prolog-like facts
RDF-like triples
JSON-like projection
phrase / synset grounding stub
```

The Prolog-like, RDF-like, and JSON-like projections are tiny adapter fixtures.
They are not full Prolog, RDF, SPARQL, JSON Canvas, or WordNet implementations.

## Authority Boundary

```text
transmutator != authority
target syntax != authority
WordNet synset != authority
Prolog fact != authority
RDF triple != authority
canonical OMI declaration + receipts = authority
```

Adapter outputs carry enough canonical OMI source information for this first
roundtrip court to prove receipt preservation. Later phases can replace that
fixture bridge with richer parsers and external adapter processes.

## Orientation-Algebra Compatibility

The OMI transmutator is compatible with Cayley-Dickson-like orientation
doubling, but it does not make that algebra an authority layer.

```text
Cayley-Dickson-like doubling = compatible orientation model
modified / split forms       = bifurcation, duality, isotropic boundary cases
trigintaduonion triads       = local incidence frames
64-dimensional continuation  = compatible with the 0x00..0x3F constitutional surface
neural / GNN / transformer   = later projection backends, not source truth
```

The relevant normalization is:

```text
weights      = projection coefficients
activations  = orientation / state readouts
triads       = local S-P-O incidence frames
receipts     = coherence witnesses
transmutator = notation / carrier / orientation preserving mapper
```

This preserves the boundary:

```text
orientation model != authority
neural backend != authority
receipt-verified OMI declaration = authority
```

> A neural transformer may become one backend of the OMI transmutator, but OMI
> transmutation is older than neural attention: it is boundary-preserving
> movement through oriented incidence frames.

## Test Coverage

Phase 95 verifies:

```text
OMI-Lisp -> S-P-O-M -> OMI-Lisp preserves identity
OMI-Lisp -> Prolog-like facts -> OMI-Lisp preserves identity
OMI-Lisp -> RDF-like triples -> OMI-Lisp preserves identity
OMI-Lisp -> JSON-like projection -> OMI-Lisp preserves identity
phrase / synset stub -> S-P-O-M grounding changes projection receipt
adapter output cannot become authority
```

## Non-Goals

Phase 95 does not:

```text
load WordNet
execute Prolog
export RDF syntax
run SPARQL
start pub/sub
perform ML inference
promote transmuted syntax to authority
```
